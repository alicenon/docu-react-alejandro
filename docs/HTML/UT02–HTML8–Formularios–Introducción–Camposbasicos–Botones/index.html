UT02 – HTML 8 – Formularios

Introducción – Campos básicos – Botones​

Lenguajes de marcas y sistemas de gestión de la información
IES Clara del Rey – Madrid

¿Qué es un formulario?​

Un formulario es una de las formas que tiene un sitio web para interactuar con sus usuarios.

Permite la recogida de datos y su envío al servidor para que sean procesados.

Ejemplo de formulario: la página inicial de Google.es.

Componentes de un formulario​

Un formulario está compuesto de:

  • Uno o más controles de formulario, como:
    • Cajas de texto (de una o de varias líneas).
    • Selectores desplegables (<select>).
    • Casillas de verificación (<checkbox>).
    • Botones de opción (<radiobutton>).
  • Contenidos dentro de un elemento <form>.
  • Un botón para enviar los datos.
  • Opcionalmente, con reglas de validación de los datos.

El elemento <form>​

Es un elemento de bloque que delimita y contiene los campos del formulario.
Principales atributos:

AtributoFunción
actionURL a la que se envían los datos del formulario.
methodMétodo HTTP (normalmente tiene de valor un POST, aunque puede ser GET).
enctypeEspecifica el formato en que se empaquetan los datos enviados al servidor mediante el método POST. Uno de sus valores, multipart/form-data, se utiliza para enviar archivos al servidor. Este valor es obligatorio cuando el formulario contiene un campo de tipo <input type="file">.

Se considera un buena práctica poner al menos un action y un method.

Si no se ponen:

  • action será la URL de la página actual (el formulario se envía a sí mismo).
  • method será GET.

El elemento <input>​

Define la mayoría de los campos de formulario. Es un elemento en línea.
Atributo principal: type (determina el tipo de campo de formulario).
Debe ir acompañado de una etiqueta <label>, que se asocia al input mediante:

  • El atributo for del <label>, que coincide con el id del <input> (mejor flexibilidad, compatibilidad, mejor control etiqueta-campo)
  • Anidando el <input> dentro del <label> (wena práctica porque mejora la accesibilidad y usabilidad)

Ejemplo sin anidar (notese el uso de for y el id con el mismo valor):

<label for="nombre">Nombre:</label>
<input type="text" id="nombre" name="nombre" />
Salida opcion 1

Ejemplo de la forma anidada:

<label>
  Nombre:
  <input type="text" name="nombre" />
</label>

Esta asociación de los inputs con sus label es fundamental para conseguir formularios accesibles.

El elemento <button>​

Se utiliza para realizar acciones en el formulario. Puede activarse mediante:

  • Ratón, teclado, táctil o comandos de voz.
  • Su aspecto base es normalmente como un botón, pero su aspecto puede cambiarse con CSS.
  • La accion al activarlo, dependerá de su tipo ('submit', 'reset')

Tipos de botones:

  • type="submit": Envía el formulario.
  • type="reset": Restaura el formulario (no recomendado).
AtributoFunción/Acción
submitEste atributo del botón envía el formulario para procesarlo en el servidor.
resetRestaura el formulario (no recomendado, porque podemos restaurar el contenido del formulario por accidente al confundirlo con el submit).

Ejemplo de botón:

<button type="submit">Enviar</button>

visualizacion:

Ejemplo de formulario simple​


<form action="recogida-datos.html" method="post">
  <p>
    <label for="input-nombre">Nombre: </label>
    <input type="text" name="nombre" id="input-nombre">
  </p>
  <p>
    <label>Apellidos:
      <input type="text" name="apellidos">
    </label>
  </p>
  <p><button type="submit">Enviar</button></p>
</form>

Método GET​

Cuando un formulario usa method="get", los datos se envían como parte de la URL. Ejemplo:

<form method="get" action="http://localhost:8080/view-form/get">
  <label>Usuario: <input type="text" name="usr"></label>
  <label>Contraseña: <input type="password" name="pwd"></label>
  <button type="submit">Enviar</button>
</form>

La URL del navegador mostrará: http://localhost:8080/view-form/get?usr=&pwd=

Método POST​

Cuando un formulario usa method="post", los datos no aparecen en la URL, sino en el cuerpo de la petición. Ejemplo:

<form method="post" action="http://localhost:8080/view-form/post">
  <label>Usuario: <input type="text" name="usr"></label>
  <label>Contraseña: <input type="password" name="pwd"></label>
  <button type="submit">Enviar</button>
</form>


Controles básicos​

Botones para enviar formularios

<button type="submit">Enviar</button>
<input type="submit" value="Enviar">
<input type="image" src="..." alt="Enviar">

Elemento <button>​

Cada campo debe tener un <label> asociado para accesibilidad. Ejemplo:

<form method="post" action="http://localhost:8080/view-form/post">
  <label>Usuario: <input type="text" name="usr"></label>
  <label>Contraseña: <input type="password" name="pwd"></label>
  <button type="submit">Enviar</button>
</form>

Asociación implícita:

<label>Usuario: <input type="text" name="usr"></label>

Asociación explícita:

<label for="usr">Usuario:</label>
<input type="text" name="usr" id="usr">

Ejemplo: Radio buttons​

No excluyentes

<input type="radio" name="opcion1" value="1"> Opción 1
<input type="radio" name="opcion2" value="2"> Opción 2

Excluyentes

<input type="radio" name="opcion" value="1"> Opción 1
<input type="radio" name="opcion" value="2"> Opción 2

Control <textarea>​

Campo para textos de varias líneas. Ejemplo:

<textarea name="comentarios" rows="4" cols="50"></textarea>

Selector <select​

Select simple:

<select name="opciones">
  <option value="1">Opción 1</option>
  <option value="2">Opción 2</option>
</select>

Select multiple:

<select name="opciones" multiple>
  <option value="1">Opción 1</option>
  <option value="2">Opción 2</option>
</select>
